Physics Coding Challenges 1: Creating Functions

This document contains a series of challenges to help you learn how to model

  1. The physics formula for acceleration is $$ a = \frac{v_f - v_i}{\Delta t} $$ Write a function named find_acceleration that takes as arguments initial_velocity, final_velocity, and time.
    find_acceleration(final_velocity, initial_velocity, time)
    If your function works well, you should be able to plug in the following values and fidn the following results:
    • find_acceleration(10,2,2) # should return 4
    • find_acceleration(2,10,2) # should return -4
    • find_acceleration(20,3,5) # should return 3.6
    • find_acceleration(0,0,0) # should return an error

    One likely problem you might have: If all of the variables entered are integers, then python will use integer division, in which is cuts off the remainder in any division problem. To sovle this problem, convert all inputs of your function to floating point values with the following commands:
    final_velocity = float(final_velocity)
    intial_velocity = float(initial_velocity)
    time = float(time)

  2. Example Momentum Problem: A cannon of mass \( M_c \) fires a cannonball of mass \( m_b \). Before the explosion, the cannon and cannonball are both stationary, afterwards the cannonball begins moving forward at a speed of \( v_b \). Determine the velocity of the cannon after the collision \( v_c \) in terms of \( M_c \), \( m_b \), and \( v_b \). Note that you should receive a negative value (assuming \( v_b \) is positive)!
  3. Write a function to find the recoil speed of a cannon after it fires a cannonball:
    find_recoil_speed(mass_of_cannon, mass_of_cannonball, velocity_of_shot)
    Note that, because you are attempting to find recoil speed, not velocity, the answer you return should be positive!
    Test your function with the following examples:
      find_recoil_speed(100,10,20) # should return 2
      find_recoil_speed(1,1,1) # should return 1
      find_recoil_speed(50.67,2.48,44.9) # should return 2.19759226
  4. Evil Knievel drives his motorcycle horizontally off of a cliff. His motorcycle is moving at a horizontal speed of \( v_x \), and the cliff has a height of \( h \). Free-fall acceleration is given by \( g \). a) Determine an equation for the time before Knievel strikes the ground in terms of \( h \), and \( g \). b) Determine how far Knievel travels on his motorcycle before striking the ground in terms of \( v_x \), \( h \), and \( g \).
  5. Create a function to determine the horizontal distance traveled when something is lunached horizontally launched off of a cliff of height h. Call your function find_horizontal_range(horizontal_speed, height, free_fall_acceleration) Test your function with the following tests:
    • find_horizontal_range(1,1,1) # should return 1.14 find_horizontal_range(20, 40, 9.81) # should return 560.3 find_horizontal_range(3.56, 8.12, 4.32) # should return 29.8
  6. Two cars get into a car crash. The first car has a mass of \( m_A \) and a velocity before the collision of \( v_A \). The second car has a mass of \( m_B \) and a velocity before the collision of \( v_B \). After the collision, the two cars stick together (this is called a totally inelastic collision). The two cars combined have a velocity of \( v_f \).
    Derive a formula for \( v_f \) in terms of \(m_A\), \(m_B\), \(v_A\), and \(v_B\).
    Something to think about: are the cars necessarily going the same direction (a rear-end)? Could they be traveling in opposite directions (a head-on collision)? How could either possibility be represented in the formula.
  7. Create a function that calculates the final velocity of two cars in a pefectly inelastic collision. find_final_velocity(mass_a, mass_b, intial_velocity_A, initial_velocity_B) Test your code with the following examples:
    • find_final_velocity(1,1,1,1) # should return 1
    • find_final_velocity(10,15,6,-5) # should return 0.2
    • find_final_velocity(8.2,4.9,-1.2,3.2) # should return 0.446

    Again, you may have problems if all of the inputs are integers and you are doing division, because Python will cut off any remainders or decimals! To fix this problem, use mass_a = float(mass_a), and likewise for all of your inputted values.
